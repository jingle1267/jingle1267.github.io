在编程这一行干活，免不了被糟糕的代码绊倒。因为糟糕的代码，导致项目延期和进度缓慢。 架构和设计模式等致力于从“战略”的角度减少糟糕代码，代码规范则侧重于从“战术”的角度减少坏代码

前言

糟糕的代码随着时间的推移对成产力的影响：
